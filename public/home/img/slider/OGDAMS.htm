    POST Authetication User
    https://ogdams.com/api/user/auth

    This platform uses Username/Password system to issue athorization tokens which are valid until till the user genrate a new one. Once you authorize to /api/auth endpoint, you will be using token to identify yourself in any subsequent queries. "Authentication" => Bearer

    Login to API (Get Token JWT) need a valid email and password pair.

    HEADERS
    Content-Type
    application/x-www-form-urlencoded
    Accept
    application/json
    Accept-Charset
    BODY raw
    {
	"email": "a@a.com",
	"password": "secret"
	
}


    Example Request
    Authetication
    curl --location --request POST "/api/user/auth" ^
--header "Content-Type: application/json" ^
--header "Accept: application/json" ^
--data "{
	\"email\" : \"a@a.com\",

    POST Perform a Airtime Topup
    /api/airtime/topup

    Perform a airtime topup to any network

    You can get the various network id by querying /api/airtime/networks endpoint:

    FieldName Type Description
    network Integer Network id of the number
    amount Integer Topup Amount
    phone String Topup phone number : 07063637002
    HEADERS
    Content-Type
    application/json
    Authorization
    Bearer <Token>
    Accept
    application/json
    BODY raw
    {
	"network" : 1,
	"amount" : 10,
	"phone" : "07063637002"
}


    Example Request
    Perform a Airtime Topup
    curl --location --request POST "/api/airtime/topup" ^
--header "Content-Type: application/json" ^
--header "Authorization: Bearer <Token>" ^
--header "Accept: application/json" ^
--data "{
	\"network\" : 1,
	\"amount\" : 10,
	\"phone\" : \"07063637002\"
}"

    POST Validate Electricity Meter
    /api/bills/electricity/2/validate

    Validate electricity meter

    This platform can validate Meter numbers or smart card numbers if service in question has parameter validation : true

    To Validate meter|smart card, perform POST query to an endpoint formatted like this : /api/bills/electricity/:serviceId/validate

    Examples :

    /api/bills/electricity/2/validate

    BODY : {"meterId" : "23300065960"}

    If validation fails or not supported, system will return HTTP 404 response.
